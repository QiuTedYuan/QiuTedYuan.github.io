4. Lower bound on finding both max and min

In the tutorial we said finding both max and min can be done in 3n/2+c comparisons. This is indeed an lower bound. Why?

Let's play a game. You design an algorithm and I will give you an input. You want to minimize the comparisons and I will maximize it. If you want to compare 2 numbers, I tell you the result. If x>y, x wins and y loses.

At any state of your algorithm, you have the following 4 kinds of numbers:

  1. a items have not been involved in any comparison.
  2. b items involved and have not lost yet.
  3. c items involved and have not win yet.
  4. d items won and lost at least 1.

Obviously a+b+c+d=n are all the numbers. My strategy is the following for each comparison.

  • (1,1): Anyway it becoms (2,3).
  • (1,2): 2 wins. So the state becomes (3,2).
  • (1,3): 1 wins, so the state becomes (2,3).
  • (1,4): Anyway it becomes (2,4) or (3,4).
  • (2,3): 2 wins, still (2,3).
  • (2,4): 2 wins, still (2,4).
  • (3,4): 4 wins, still (3,4).

The logic is that: if you haven't won, then I try not to let you win. If you havn't lost, I try not to let you lose. This will maximize the comparisons. Consider the magic number 3a+2b+2c. At the beginning it is equal to 3n, when your algorithm finishes it has to be 2. Check that each of the above comparison decrease this number at most 2 at a time (first case). A total decrease of (3n-2) requires at least 3n/2-1 comparisons.

(Randomly or deterministicly) Generate permutations of the input until it finds one that is sorted.

2. Sleep Sort

Suppose you have abundant processors. Let each processor hold a number "x". With all starting at the same time, this processor will "sleep" for x rounds. It immediately outputs the number after it "wakes up".

The running time "is" (not rigorous) O(n+max{input}).

4. Egg Dropping Puzzle

As a special case, suppose there are 2 eggs and a 100-floor building. You want to test from which floor on the egg will break when you drop it to the ground.

If you think of doing a "binary search", the problem is, if the egg broke on the 50-th floor, you have to go one-by-one for the second egg, the total number becomes 1+49=50 in the worst case.

The idea is to divide the 100 floors into sqrt(100)=10 blocks, each containing sqrt(100)=10 floors. Use the first egg to locate the block and the second to locate the exact floor. The worst case is then 10+10=20.

Challenge: What if there are more floors? More eggs?

There is also a DP solution which you can find in the link.

5. Secretary Problem

You are going to hire exactly 1 secretarty out of n applicants that you can rank between. Each applicant will be interviewed one by one and you must decide whether to hire her before interviewing the next one. You want to maximize the probability that you hire the best of the n applicants.

The optimal strategy is to skip the first ~n/e applicants and pick the first applicant that outperforms all the skipped ones.

If you are the best, try not to be the first ~n/e in this interview XD.

This problem is closely related to the hiring problem.

6. 0-1 Knapsack Problem

You are given n items, each having value vi and weight wi. You are also given a knapsack that can hold at most W units of weight. You want to select some items and maximize the value you get. This problem is interesting because:

1, Greedy algorithms can not solve the problem. Please think of counter examples for the following algorithms:

  1. Always pick the most valuable item possible. (max vi)
  2. Always pick the lightest item possible. (min wi)
  3. Always pick the heightest "unit value" item possible. (max vi/wi)

2, There is a Dynamic Programming Solution to the problem, which runs in O(nW).

Question: Is this a polynomial time algorithm?

Answer: No! To input the number W, you only need Theta(log(W)) bits. Thus the input size is (n+log(W)). The factor W is expotional in input size.

3, It is actually an NP-Complete problem! The decisional version of the problem is:

Given the problem setting, is there a valid set of items that has value at least V?

It is NP because if there is a solution, we can calculate its value and weight in O(n) time and make sure it satisfy the requirements.

It is NP-Hard because it can be reduced from the "number partitioning" problem. Assign each item (vi,wi)=(xi,xi). Set both the weight constraint and value target to be W=V=sum(xi)/2.

8. Matrix Multiplication

An very interesting and important problem. The naive divide and conquer algorithm can divide it into 8 subproblems, thus run in O(n^3).

Strassen's algorithm divides the problem into 7 subproblems instead. The power factor is reduced to log2(7)=2.8. Currently the best algorithm achieves n^2.37, but the conjecture is that this is still not the best.
